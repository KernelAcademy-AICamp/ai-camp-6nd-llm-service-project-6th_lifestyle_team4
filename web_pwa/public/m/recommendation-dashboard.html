25칸 중 20칸만 사용 · 최다 = 차갑고 격렬(45장)
적음많음행=온도 · 열=강도

④ 왜 약한가 — 가중 곡선 1/(1+d)

거리가 멀어져도 가중치가 충분히 안 떨어진다. 최악 매칭 카드도 최선의 15% 확률 → 추천이 거의 평평
개인화 강도 측정: 취향과 가장 가까운 상위 10% 카드가 가져가는 선택 확률이 22%에 불과(균등이면 10%) → 랜덤 대비 약 2.2배. 제안의 softmax(τ=0.5)로 바꾸면 이 집중도를 원하는 만큼 키울 수 있다.

⑤ 제안 점수식 TO-BE

온보딩(장르·주제) + 행동(온도·강도) + 인기 가중 합 → softmax 샘플링
score(c) =
  wg·genreMatch + wt·themeMatch
  + wb·tasteSim + wp·popularity
P(c) ∝ exp( score(c) / τ )
신호정의floor
genreMatch선택 장르면 10.15
themeMatch선택 주제면 1 / "상관없음"=10.2
tasteSim1 − d/√32 (북마크≥10)0
popularity북마크수 log 정규화—

⑥ 콜드스타트 → 행동 전환 TO-BE

α = min(북마크/10, 1). 가입 직후엔 온보딩 100%, 쌓일수록 행동 비중 ↑
가입 직후(α=0): 장르·주제만으로 추천 → 신규 유저 콜드스타트 해결. 북마크 10개(α=1): 온보딩 60% + 행동 35% + 인기 5%.

⑦ 적용 로드맵

  • P0 온보딩 선호 수집(장르·주제) 저장 + 카드 테마 컬럼화(키워드→10범주 배치). 선결
  • P1 genreMatch+themeMatch 점수 추가, blend(α) 도입, 기본 ON. → 콜드스타트 해결·체감 큰 변화
  • P2 softmax(τ) 샘플링, 인기도 약가중, 새로고침=부정신호(세션 감점)
  • P3 KPI 측정·튜닝: 추천 클릭률·북마크 전환율·세션당 새로고침수·테마 커버리지 / A·B 테스트
